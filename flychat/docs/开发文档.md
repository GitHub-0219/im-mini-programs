# FlyChat 开发文档

## 1. 技术架构

### 1.1 整体架构

```
┌─────────────────────────────────────────┐
│              前端展示层                    │
│  HTML + CSS + Vanilla JavaScript         │
├─────────────────────────────────────────┤
│              数据层                       │
│  Mock Data (js/data.js)                  │
├─────────────────────────────────────────┤
│              样式层                       │
│  CSS Variables + Responsive Design       │
└─────────────────────────────────────────┘
```

### 1.2 文件结构

```
flychat/
├── index.html          # 消息列表页
├── chat.html           # 聊天页
├── contacts.html       # 通讯录页
├── workspace.html      # 工作台页
├── profile.html        # 个人中心页
├── group.html          # 群组管理页
├── css/
│   └── style.css       # 全局样式（CSS变量、响应式）
└── js/
    └── data.js         # 模拟数据（FlyChat对象）
```

### 1.3 核心设计模式

- **CSS变量主题系统**：通过 `:root` 定义颜色、间距等变量，方便主题切换
- **响应式断点**：Mobile(<768px) / Tablet(769-1199px) / PC(≥1200px)
- **组件化思路**：通过CSS class复用样式，页面间通过URL参数传递数据
- **数据驱动**：FlyChat全局对象管理所有模拟数据

## 2. 接口说明（模拟）

当前为纯前端演示，所有数据通过 `js/data.js` 中的 `FlyChat` 对象提供。

### 2.1 数据结构

#### 会话（Conversation）
```javascript
{
  id: string,          // 会话ID
  name: string,        // 会话名称
  avatar: string,      // 头像文字
  type: 'private'|'group', // 类型
  lastMsg: string,     // 最后消息
  time: string,        // 时间
  unread: number,      // 未读数
  pinned: boolean,     // 是否置顶
  muted: boolean       // 是否免打扰
}
```

#### 消息（Message）
```javascript
{
  id: string,          // 消息ID
  sender: string,      // 发送者ID
  name: string,        // 发送者名称
  avatar: string,      // 头像
  type: 'text'|'image'|'voice'|'file', // 消息类型
  content: string,     // 内容
  time: string,        // 时间
  status: 'sent'|'delivered'|'read', // 状态
  atUser?: string,     // @的用户
  replyTo?: string,    // 引用的消息ID
  fileSize?: string,   // 文件大小
  duration?: string    // 语音时长
}
```

#### 联系人（Contact）
```javascript
{
  id: string,          // 联系人ID
  name: string,        // 姓名
  avatar: string,      // 头像
  title: string,       // 职位
  dept: string         // 部门
}
```

### 2.2 接口预留（后端对接）

```javascript
// 消息相关
GET    /api/conversations           # 获取会话列表
GET    /api/conversations/:id/messages  # 获取消息
POST   /api/conversations/:id/messages  # 发送消息
PUT    /api/conversations/:id/read      # 标记已读
DELETE /api/messages/:id                # 撤回消息

// 联系人相关
GET    /api/contacts                # 获取联系人列表
GET    /api/contacts/:id            # 获取联系人详情
GET    /api/org/tree                # 获取组织架构

// 群组相关
GET    /api/groups/:id              # 获取群信息
GET    /api/groups/:id/members      # 获取群成员
GET    /api/groups/:id/files        # 获取群文件
POST   /api/groups/:id/announcements # 发布群公告
```

## 3. 数据库设计（参考）

### 3.1 核心表

```sql
-- 用户表
CREATE TABLE users (
  id VARCHAR(32) PRIMARY KEY,
  name VARCHAR(64) NOT NULL,
  avatar VARCHAR(256),
  title VARCHAR(128),
  department_id VARCHAR(32),
  phone VARCHAR(20),
  email VARCHAR(128),
  status ENUM('online','busy','away','dnd') DEFAULT 'online',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 会话表
CREATE TABLE conversations (
  id VARCHAR(32) PRIMARY KEY,
  type ENUM('private','group') NOT NULL,
  name VARCHAR(128),
  avatar VARCHAR(256),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 会话成员表
CREATE TABLE conversation_members (
  conversation_id VARCHAR(32),
  user_id VARCHAR(32),
  role ENUM('owner','admin','member') DEFAULT 'member',
  is_pinned BOOLEAN DEFAULT FALSE,
  is_muted BOOLEAN DEFAULT FALSE,
  unread_count INT DEFAULT 0,
  joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (conversation_id, user_id)
);

-- 消息表
CREATE TABLE messages (
  id VARCHAR(32) PRIMARY KEY,
  conversation_id VARCHAR(32) NOT NULL,
  sender_id VARCHAR(32) NOT NULL,
  type ENUM('text','image','voice','file','system') NOT NULL,
  content TEXT,
  file_url VARCHAR(512),
  file_size BIGINT,
  reply_to_id VARCHAR(32),
  at_users JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_conversation (conversation_id, created_at)
);

-- 消息状态表
CREATE TABLE message_status (
  message_id VARCHAR(32),
  user_id VARCHAR(32),
  status ENUM('sent','delivered','read') DEFAULT 'sent',
  read_at TIMESTAMP NULL,
  PRIMARY KEY (message_id, user_id)
);

-- 组织架构表
CREATE TABLE organizations (
  id VARCHAR(32) PRIMARY KEY,
  parent_id VARCHAR(32),
  name VARCHAR(128) NOT NULL,
  type ENUM('company','dept','team') NOT NULL,
  sort_order INT DEFAULT 0
);

-- 群公告表
CREATE TABLE announcements (
  id VARCHAR(32) PRIMARY KEY,
  group_id VARCHAR(32) NOT NULL,
  content TEXT NOT NULL,
  author_id VARCHAR(32) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 部署方案

### 4.1 静态部署

```bash
# Nginx配置
server {
    listen 80;
    server_name flychat.example.com;
    root /var/www/flychat;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 开启gzip
    gzip on;
    gzip_types text/css application/javascript;
}
```

### 4.2 CDN部署

将 `css/`、`js/` 静态资源上传至CDN，HTML中引用CDN地址。
