# FlyChat 交付文档

## 1. 部署说明

### 1.1 环境要求

- Web服务器：Nginx / Apache / 任意静态文件服务器
- 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 无需Node.js、数据库或其他后端服务

### 1.2 部署步骤

#### 方式一：Nginx部署

```bash
# 1. 复制项目文件到Web目录
cp -r flychat/ /var/www/flychat/

# 2. 配置Nginx
cat > /etc/nginx/sites-available/flychat << 'EOF'
server {
    listen 80;
    server_name flychat.example.com;
    root /var/www/flychat;
    index index.html;

    # gzip压缩
    gzip on;
    gzip_types text/css application/javascript text/html;
    gzip_min_length 1000;

    # 缓存策略
    location ~* \.(css|js)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
EOF

# 3. 启用站点
ln -s /etc/nginx/sites-available/flychat /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

#### 方式二：Docker部署

```dockerfile
FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80
```

```bash
docker build -t flychat .
docker run -d -p 8080:80 flychat
```

#### 方式三：本地开发

```bash
# Python
python3 -m http.server 8080

# Node.js
npx serve .

# 直接打开
open index.html
```

### 1.3 HTTPS配置

```nginx
server {
    listen 443 ssl;
    server_name flychat.example.com;

    ssl_certificate /etc/letsencrypt/live/flychat.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/flychat.example.com/privkey.pem;

    root /var/www/flychat;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

server {
    listen 80;
    server_name flychat.example.com;
    return 301 https://$server_name$request_uri;
}
```

## 2. 运维手册

### 2.1 日常运维

#### 日志查看
```bash
# Nginx访问日志
tail -f /var/log/nginx/flychat_access.log

# Nginx错误日志
tail -f /var/log/nginx/flychat_error.log
```

#### 健康检查
```bash
# 检查服务状态
curl -s -o /dev/null -w "%{http_code}" https://flychat.example.com/

# 检查响应时间
curl -s -o /dev/null -w "Time: %{time_total}s\n" https://flychat.example.com/
```

### 2.2 更新部署

```bash
# 1. 备份当前版本
cp -r /var/www/flychat /var/www/flychat_backup_$(date +%Y%m%d)

# 2. 部署新版本
cp -r flychat/* /var/www/flychat/

# 3. 清除CDN缓存（如使用CDN）
# 4. 验证部署
curl -s https://flychat.example.com/ | grep -o '<title>.*</title>'
```

### 2.3 监控指标

| 指标 | 阈值 | 告警方式 |
|------|------|----------|
| HTTP状态码 | 非200 | 邮件/短信 |
| 响应时间 | >3s | 邮件 |
| 可用性 | <99.9% | 邮件/电话 |
| 磁盘使用 | >80% | 邮件 |

### 2.4 故障处理

| 故障现象 | 可能原因 | 处理方案 |
|----------|----------|----------|
| 页面空白 | 资源加载失败 | 检查CDN/服务器、清除缓存 |
| 样式错乱 | CSS文件损坏 | 重新部署CSS文件 |
| JS报错 | 浏览器兼容性 | 检查浏览器版本、降级处理 |
| 404错误 | 路由配置错误 | 检查Nginx try_files配置 |

### 2.5 备份策略

- **代码备份**：Git版本控制，每日自动备份
- **静态资源**：CDN多副本冗余
- **配置备份**：Nginx配置纳入Git管理

## 3. 安全建议

- 启用HTTPS
- 配置CSP（Content Security Policy）
- 开启X-Frame-Options防嵌入
- 定期更新服务器系统和软件
- 监控异常访问日志
